APML at Idiomag

I saw over at the Read/Write Web that Idiomag is now supporting APML.  In theory, you can give Idiomag a URL to your APML file and it will bulid a music magazine for you with articles, reviews, galleries and interactive features.   Since I happen to have created a web service that will generate an APML file based upon my music listening behavior (see tastebroker.org) , I thought I give it a try.   I plugged in the URL (  http://TasteBroker.org/apml/last.fm/lamere ). But alas, it didn't work - I received the Idiomag error page (Oops! Something seems to have gone wrong!).  

Perhaps the folks over at IdioMag can help drop me a line so we can figure out where things are going awry - what good is a standard for taste portability if it isn't really portable!

Update:  Ah, I forgot that 'tastebroker.org' was just a redirect to http://research.sun.com:8080/AttentionProfile.  If I use the true APML link of http://research.sun.com:8080/AttentionProfile/apml/last.fm/lamere everything works fine.  (and Ed Barrow from IdioMag comments that they've updated Idiomag to deal with the redirect as well.  Now that is nimble!)
